Node.js Study Guide

Node.js is based on asynchronous event-driven architecture and non-blocking Input/Output API. Provides a way for web applications to have real-time 2 way connections between client and server where either side can initiate communication and data can be exchanged freely. The Node.js library uses JavaScript. Used by large companies like Paypal, Linkedin, Mozilla and eBay.

Best Use Cases: Streaming, or event-based real-time applications like the following examples:

  • Chat applications
  • Game servers
  • Collaborative environments (ex. Managing documents with multiple people)
  • Advertisement servers
  • Streaming servers
  • Client side-applications which are based on same javascript framework

Concurrent request handling means if a request is received by Node for an input or output operation, it will go ahead and execute while still running other requests at the same time in the background. For example, with readFile, other programming languages would not move on until the whole file was read. With Node, the file is being read in the background while other things are happening too.

Before Node.js and concurrent request handling, what existed was called a "Stateless Model." A stateless model is where data generated in one session is not maintained for usage in the next session with that user

The callback function is the asynchronous equivalent of a function. Callback functions are used heavily in Node.js. The callback function follows a simple structure:

('function(error,data)')

The callback function uses non-blocking code, which means that the program doesn't wait for the file reading to finishs before completing the desired output. For example, the callout function may print "Program Ended" in the console.log before the program has finished reading the file. The asynchronous nature makes Node.js very scalable. It can process a high number of requests without having to wait for each step of previous code to complete.

The error section of the callback function is designed to catch and alert developers of any potential errors created by the process of reading the file. It's a "catch-all" error alert, where otherwise devlopers may not be aware there is an issue, or what that issue is. It is always included and can technically be named anything, however, error, or err are most commonly seen.

The require function reads the Javascript file, executes the file and then returns an object. It is the easiest way to include modules that exist in separate files. The require function is used for built-in core Node.js modules, community-based modules and local/custo modules. The require function follows a simple structure:

var example = require('./example.js')

Modules allow for logical encapsulation of code in a single unit. Best practices include segregateing code to make it most manageable for the future, and modules help allow for that. Each module is an independent entity and can be managed as a separate unit of work. There are many premade modules that can be used with Node.js, like Express framework, Socket.io, Jade, MongoDB, Restify, Bluebird. Developers can also create custom modules.

Before using premade or existing modules, they need to be installed using the Node Package Manager (NPM). When you install a premade module correctly, you get all the necessary files that you need to use it moving forward within your application. To install a module you use the simple install command:

npm install express

Express represents the example module name

After you install you need to use the require keyword so that Node.js knows to incorporate the functionality of the module. Below is an example of using the require keyword:

var express=require(‘express’);

This is using the require keyword

Var app=express();

This is creating an object of express

app.set(‘view engine’, ‘jade’);

This is the calling the set method

Var server = app.listen(3000, function () { });

This is calling the listen method, which makes the application listen on a particular port number

You can also create custom modules. This is basically like creating a function that you can run based on user inputs. After creating the custom module, you must create an application which will call this module into action. Custom modules do not require the Node Package Manager installation because they are already part of your project on your local machine. Below is an example of creating a custom module:

Var exports = module.export={ };

This is defining the module

exports.AddNumber=function(a,b){

This is creating the function in our module

Return a+b; };

This is asking the function to return a value back to the calling function

To get the custom module to run, you’ll need to create a function to call it:

Var Addition=require(‘./Addition.js’);

Here we are using require to include our custom module. We must include the require BEFORE passing any parameters

console.log(Addition.AddNumber(1,2));

Here we are calling the function with specific arguments to display in the console
